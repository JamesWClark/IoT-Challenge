Step 1: Create a WebClient folder in the root of your team project files.


Step 2: Your server team should have their own root folder, and so should your sensors team. Each team should separate their work into smaller sub projects. If not, take some time with your team and clean things up a bit. As a team, you should be managing a root project similar to the above screenshot.

Step 3: In the WebClient folder, create an index.html file. This is the home page for your project. For now, let's keep it simple:
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Document</title>
</head>
<body>
  <h1>Site Under Construction</h1>
</body>
</html>
Step 4: Your server team is writing a server script for Node.js and you need to suggest a configuration change to their system. In your team's project files, locate their server script and find the following snippet of code:
app.get('/', function (req, res) {
    res.send('Hello World!');
});
Step 5: Your server team was previously instructed to include that code and you need to ask them to delete it. You could do it yourself, but that's not your concern. You are the clients team. Give your server team the following replacement code and tell them it's middleware and should go immediately after the body parsing middleware. If they ask why, you can tell them it's necessary for Node.js to act as a static file web server which you are responsible for managing.
app.use('/', express.static('../WebClient/'));
Step 6: Pull the server team's changes into your local repository. If this is your first time running the server, you probably need to install node packages.
npm install
Step 7: Run their server script. Vist the "Site Under Construction" page you are hosting to verify everything's working correctly so far: http://localhost:3000/
node server

Step 8: Before continuing on, notify your instructor when you've completed this task.
